## 3. Memory Garden (lume nouă + componentă nouă)

### Componentă: [`src/components/game/MemoryMode.tsx`](src/components/game/MemoryMode.tsx)

State machine: `'show' → 'input' → 'reveal'` (sau retry loop).

```
[show] → cells filled with target pattern, no interaction, ~3s
   ↓ showDurationMs timeout
[input] → cells cleared, child taps to reproduce
   ↓ pattern matches (1.5s hold)        ↓ overshoot/wrong (1.8s grace)
[reveal] → original pattern + voice    → re-show pattern, back to [input]
```

### Pattern match strict
**Important:** match = userCells positions === targetCells positions. Nu doar count match.

```ts
const patternMatches = userCells.every((cell, i) =>
  (cell !== 'empty') === (challenge.targetCells[i] !== 'empty')
);
```

User a vrut clar pattern match strict (testat user a zis că count-only era prea lenient — putea pune 4 cells în pozitii greșite și să fie „corect").

### Niveluri progresive
Definit în [`mathProblems.ts` generateMemoryChallenge](src/utils/mathProblems.ts):

| modeLevel | targetCount range | showDurationMs |
|---|---|---|
| 1 | 1-2 | 3000ms |
| 2 | 2-3 | 2500ms |
| 3 | 3-4 | 2200ms |
| 4 | 4-5 | 2000ms |
| 5 (champion) | 5-7 | 1500ms |

### Voice clips (Memory)
- `mem_watch`: „Watch carefully!"
- `mem_count`: „Now your turn!"
- `mem_was_N` (N=1-7): „There was 1!" / „There were N!"

Voice triggered prin `onPhaseChange` callback din MemoryMode către AdventureLevelScreen.

### Layout stabil
**Bug fix:** counter view aparea doar în `input` phase → layout shift între faze. Fix: `counterSlot` cu `height: 70` fixă, content opțional înăuntru.

### Retry logic
- **Wrong (first time)**: `onWrong()` increment attempts + voce „Try again!", apoi 1.8s grace, apoi re-show pattern + retry input. Nu avansează la următoarea provocare.
- **Wrong (subsequent)**: doar grace + re-show (nu re-incrementează attempts, `hadWrongRef` latch).
- **Correct după wrong**: `onCorrect` cu `wasFirstTry=false` → records as miss → stele mai puține.

## 4. Auto-submit (no ✓ button)

### Free Play (already from v1.6 P1)
[CountingMode](src/components/game/CountingMode.tsx), [AdditionMode](src/components/game/AdditionMode.tsx), [SubtractionMode](src/components/game/SubtractionMode.tsx) folosesc auto-submit pentru profile young.

### Adventure (nou în această sesiune)
[`AdventureLevelScreen`](src/components/adventure/AdventureLevelScreen.tsx) auto-submit useEffect care detectează:
- counting fill_exactly: `totalFilled === targetNumber`
- counting fill_top_row: `top=5 && bottom=0`
- counting fill_bottom_row: `bottom=5 && top=0`
- counting fill_both_equal: `top=N/2 && bottom=N/2`
- puzzle make-10: `color2Count === 10-num1`
- addition: `color2Count === num2`
- subtraction: `remainingColor1 === answer`

Timing:
- 350ms la match (positive feedback rapid)
- 1800ms la overshoot grace (wrong feedback după)

Manual ✓ button scos peste tot (memory are propria mecanică prin MemoryMode).

### Latch reset pe retry
Bug subtil: după wrong submit, dacă copilul tap-ează un cell (`handleCellPress` setează `hasSubmitted=false`), auto-submit nu mai fire-ea fiindcă `autoSubmitFiredRef.current` rămânea `true`. Fix: useEffect care resetează latch când `hasSubmitted` devine `false`.

## 8. Bugs majore rezolvate

### Voice timer cancellation pe re-render
**Problem:** useEffect-uri cu callbacks (`onCorrect`, `onWrong`, `voice.play`) în deps. Parent re-renders frequent → effects refire → cleanup `clearTimeout` cancel timer înainte să fire.

**Fix universal:** ref pattern pentru callbacks în deps:
```tsx
const voiceRef = useRef(voice);
voiceRef.current = voice;
// useEffect deps: [visible] doar, NU [visible, voice]
```

Aplicat în: PlayerSetup, ModeChoice, AdventureMapScreen, AdventureLevelScreen, MemoryMode.

### Voice overlap (result + new problem)
**Problem:** după răspuns corect, voce `post_great_*` (~2s) joacă, dar imediat e schimbat problemul → voce intro nou problem începe → overlap.

**Fix:** delay 1.6s pe new problem voice (în AdventureLevelScreen useEffect on currentProblem change).

### Voice double-fire pe problem change
**Problem:** useEffect cu `[currentProblem, problemIndex, ...]` deps fire de 2 ori — odată cu currentProblem vechi (pe problemIndex change), apoi cu fresh after state update. Vechea voce și noua se suprapun.

**Fix:** scos `problemIndex` din deps. Key dedup bazat pe content (num1, num2, instruction).

### `filledCount` în Adventure ignora `color1`
**Problem:** NumberDisplay arăta 0 pentru subtraction (cells rămase sunt color1, nu erau numărate).

**Fix:** `cells.filter(c => c !== 'empty').length` (count toate non-empty).

### Memory advance după wrong (latch stuck)
**Problem:** după wrong auto-submit + user retry, `autoSubmitFiredRef` rămânea `true` → nu mai fire-ea correct path → joc blocat.

**Fix:** useEffect care resetează ref când `hasSubmitted` devine `false`.

### Memory count-match prea lenient
**Problem:** user putea pune 4 cells în orice poziții (target=4) și era corect.

**Fix:** pattern match strict — fiecare cell trebuie să corespundă pozițional cu pattern-ul show.

### Memory correct lock-in prea rapid
**Problem:** correctFiredRef setat eager înainte de timer 350ms. Dacă copilul tap-ează rapid și schimbă state, timer cancelled dar ref stays true → joc blocat.

**Fix:** ref latched DOAR în timer callback (cand actually fires). Plus timer extended 350ms → 1500ms (copilul are timp să verifice).

### WorldSelector overflow vertical
**Problem:** după convertire la ScrollView horizontal, tab-urile lumilor se făceau verticale uriase (1/2 ecran).

**Fix:** `style={{flexGrow: 0}}` pe ScrollView.

### Diferențe potențiale Android vs iOS

#### Audio
- iOS: `Sound.MAIN_BUNDLE`, files în root bundle
- Android: `res/raw/`, file names trebuie să fie **lowercase + alfanumerice + underscore** (deja respectat: `voice_en_num_1.mp3`)
- `react-native-sound` are același API pe ambele

**Verificare**: play voice la pornirea app pe Android, ascultă dacă se aude.

#### Font Fredoka
- iOS: declarat în `Info.plist` UIAppFonts: `Fredoka.ttf`
- Android: file e în `android/app/src/main/assets/fonts/Fredoka.ttf`, dar **font family name** trebuie să fie identic cu PostScript name din font
- Verificare: textul ar trebui să apară cu Fredoka. Dacă fallback la sistem, recreate cu numele corect

#### Shadows și elevation
- iOS folosește `shadowColor`, `shadowOffset`, `shadowOpacity`, `shadowRadius`
- Android folosește `elevation` (deja setat în butoanele cheie)
- Verificare vizuală: shadow-urile pe submit button, theme cards, etc.

#### Back button hardware
- iOS nu are buton fizic înapoi → modal close prin ✕
- Android are buton hardware back → trebuie să închidem modal-urile corect
- `Modal` component din React Native are `onRequestClose` callback — deja setat în AdventureMapScreen
- **Verifică toate modal-urile**: PlayerSetup, ModeChoice, AdventureMapScreen, AdventureLevelScreen, AboutTenFrames, UpgradeScreen, DailyLimitModal, StickerBook, AchievementsScreen, NewStickerPopup, AchievementPopup, MilestoneAnimation

#### Status bar
- iOS: respectă `translucent` prop, ajustăm `paddingTop: 56` în headers
- Android: poate avea status bar diferit (`StatusBar.setBackgroundColor` necesar)

#### Performance Reanimated
- Reanimated v4 + worklets necesar pe ambele. Verifică că Gradle build trece.

### Build Gradle versions
- Folosim Gradle 8.13 + JDK 17 (vezi CLAUDE.md). Gradle 9.0 incompatibil cu plugins curent.

### IAP

- Android: necesită configurare separată în Google Play Console (produs `unlock_premium`)
