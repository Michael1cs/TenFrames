# v1.6 Accessibility Changes — Apply to iOS

**Context:** Teacher/specialist feedback on v1.4.0 flagged 11 areas for improvement (grammar, text amount, legibility, sound quality, engagement, creativity, etc.). v1.5 Adventure Map addressed replayability/engagement. **v1.6 addresses the remaining accessibility gaps, primarily for pre-readers aged 5-7.**

Branch: `feature/v1.6-accessibility`

---

## What changed (and why)

### 1. Grammar & spelling consistency (RO)
**Files:** `src/i18n/locales/ro.json`, `src/i18n/locales/en.json`

- `youHave`: `"Ai {{count}} bilețele!"` → `"Ai {{count}} bile!"` — the rest of the file uses "bile" consistently; the diminutive form was an inconsistency that a teacher reviewer could flag.
- **RO `whatDesc`** had a grammar issue: "Dezvoltat ca material didactic... Ten Frames este" — the verb form and subject agreement were awkward. Replaced with shorter, cleaner version.
- **EN `whatDesc`** rewritten from 110 words → ~35 words.

### 2. Simplified About-modal texts (removed pedagogical jargon from child-visible sections)
Same files as above. Sections: `info.whatDesc`, `info.howHelpDesc`, `info.countingDesc`, `info.additionDesc`, `info.subtractionDesc`, `info.bondsDesc`, `info.benefitsDesc`.

Removed from child-visible paths:
- `"subitizing"` / `"subitizare"` → "recognize without counting" / "recunoaște rapid"
- `"CPA approach"` → removed entirely (too jargony)
- `"addend"` / `"termen"` → plain language ("first number", "primul număr")
- `"complements of 10"` / `"complementele lui 10"` → "pairs that make 10" / "perechile care fac 10"

Kept in `info.researchDesc` (parents-only section): NCTM, Common Core, Singapore Math — these are legitimate references for parents but should not be the first thing a 5-year-old sees.

### 3. Legibility — font sizes increased
**File:** `src/components/layout/ModeSelector.tsx`

| Where | Before | After |
|---|---|---|
| `bottomLabel` (tab text portrait) | 11 | 14 |
| `labelVertical` (tab text landscape) | 13 | 15 |
| `remainingBottom` (e.g. "3/5") | 9 | 12 |
| `remainingVertical` | 10 | 12 |
| `remainingPlaceholder` | 9 | 12 |

**Files:** `src/components/game/AdditionMode.tsx`, `SubtractionMode.tsx`, `PuzzleMode.tsx`

| Where | Before | After |
|---|---|---|
| `hint` (instruction below problem) | 13 | 16 |
| `feedbackText` (correct/wrong banner) | 14 | 16 |
| `answerText` | 13 | 15 |
| PuzzleMode `question` | 14 | 16 |

### 4. Legibility — contrast fix
`src/components/layout/ModeSelector.tsx` — inactive tab labels were `rgba(255,255,255,0.85)` on dark backgrounds (marginal WCAG AA). Now pure `#FFFFFF` with `fontWeight: '600'` for non-active state, `'bold'` for active. Distinction now comes from weight, not opacity.

### 5. Voice narration for pre-readers (new)
This is the biggest accessibility win. Many 5-year-olds can't read instructions.

**Package added:** `react-native-tts@^4.1.1`

**New files:**
- `src/hooks/useNarration.ts` — exposes `speak`, `speakKey`, `stop`; respects audio mode.
- `src/components/common/AudioToggle.tsx` — 3-state cycle button (🔊 full / 🔈 sfx-only / 🔇 mute).

**Type added:** `AudioMode = 'full' | 'sfx' | 'mute'` in `src/types/game.ts`. Optional `audioMode` field on `PlayerData`, persisted via AsyncStorage.

**Wired into `GameShell.tsx`:**
- On app load (player data loaded): reads `audio.welcomeBack` with kid's name after 800ms.
- On mode change (Addition/Subtraction/Puzzle): reads the instruction once.
- On correct answer: reads `audio.correctCheer` ("Bravo!" / "Well done!").
- On wrong answer: reads `audio.wrongCheer` ("Încearcă din nou!" / "Try again!").

**`useSound` was extended** to accept `audioMode` — muted mode also silences SFX. The hook signature changed from `useSound()` → `useSound(audioMode?)`; existing callers without args still default to `'full'`.

### 6. Animation duration — snappier celebrations
Reviewers noted 2.5–3.5s celebrations are too long for 5-year-olds (attention drifts). Now 1.5–2s.

- `src/components/feedback/CorrectAnimation.tsx`: confetti piece duration 2500+1000 → 1400+400ms; fade delay 2000 → 1100ms; fade duration 500 → 300ms; random start delay max 2000 → 800ms.
- `src/hooks/useGameState.ts`: three `setTimeout(() => setShowConfetti(false), 3000)` calls → 1800ms (level-up, correct answer, puzzle solved); wrong-retry timeout 3000 → 2000ms.

### 7. i18n keys added
New `audio.*` namespace in both `ro.json` and `en.json`:
- `modeFull`, `modeSfx`, `modeMute` — toggle labels (accessibility label for screen readers)
- `welcomeBack` — "Bună, {{name}}!" / "Hi, {{name}}!"
- `instructionAddition`, `instructionSubtraction`, `instructionPuzzle`, `instructionCounting` — narrated instructions
- `correctCheer`, `wrongCheer` — narrated feedback

### 8. Version bump
- `android/app/build.gradle`: `versionCode 9 → 10`, `versionName "1.4.0" → "1.6.0"`
- `package.json`: `"version": "1.5.0" → "1.6.0"`

## iOS-specific steps

These are the things I could NOT do from this machine (no iOS build chain here).

### A. Install native iOS dependency for react-native-tts

After `git pull` on the iOS machine:

```bash
cd /path/to/Tenframes
npm install                 # pulls react-native-tts from package.json (already committed)
cd ios
pod install                 # links TTS native module into the iOS project
```

### B. Info.plist — add speech permission string

`ios/TenFrames/Info.plist` likely needs a usage description key so iOS doesn't reject the speech API. Verify the following entry exists (add if missing):

```xml
<key>NSSpeechRecognitionUsageDescription</key>
<string>Ten Frames reads instructions aloud to help young learners.</string>
```

Note: `react-native-tts` uses `AVSpeechSynthesizer` which generally does NOT require permission on iOS. But some App Review reviewers flag this, so it's safer to add the key. If you want to be strict-minimal, test without it first and only add if Apple rejects.

### C. Bump iOS version

`ios/TenFrames.xcodeproj/project.pbxproj` or via Xcode UI:
- `CFBundleShortVersionString` → `1.6.0`
- `CFBundleVersion` (build number) → increment from whatever v1.5 build 3 was (so `4` or higher)

Alternatively edit `ios/TenFrames/Info.plist` directly if the versions live there.

### D. Test narration on iOS device

iOS TTS voices:
- RO may fall back to system if no Romanian voice is installed. On device: Settings → Accessibility → Spoken Content → Voices → Romanian — download "Ioana" if missing.
- EN always works out of the box.

Test the 3-state `AudioToggle`:
1. Open app — you should hear "Hi, [name]!" after ~1s.
2. Switch to Addition — you should hear "Tap empty cells to add dots!"
3. Answer correctly — "Well done!"
4. Tap 🔊 → becomes 🔈 → SFX only, no voice.
5. Tap 🔈 → becomes 🔇 → silent.
6. Tap 🔇 → cycles back to 🔊.
