# Session Notes — 2026-05-11/12

Summary of work shipped across this two-day session, plus what's still open
for the next iteration.

## Done

### New world / mode: Farm Share
Replaced the abstract "Divide Tower" decomposition mode with a fair-share
mechanic that maps better to 4-6 year olds.

- New `share` GameMode + `farm-share` WorldId.
- New `FarmShareMode` component: N pieces of food in a ten-frame pool, K
  animal baskets below; tap a basket (or a food in the pool) to drop one
  in; `−` button below each basket to take back.
- 8 levels: ÷2 progression (4→10), ÷3 (6/9), ÷5 (10, maps to ten-frame rows),
  + 2 bonus.
- Fixed food/animal pair per level so a level called "Bunnies & Carrots"
  always shows 🐰/🥕 across all 5 problems.
- Per-level basket density (roomy / compact / tiny) at fixed widths so the
  basket frame never shifts as food is added/removed, wraps naturally
  (3 baskets → 2+1, 5 baskets → 3+2).
- Validation colors only when the pool empties: green = correct portion,
  red = too many (and only on levels 1-2 as training wheels — later levels
  rely on voice alone).
- Pool ten-frame renders the actual food emoji (🥕, 🌽, 🍎, 🌾) instead of
  the theme's red marble.
- Voice clips: `share_intro`, `share_again`, `share_unfair`, `world_farm_share`.

### Memory mode polish
- 8s inactivity timer in input phase — re-shows the pattern automatically
  if the child froze. Counts as an attempt.
- Memory pattern no longer leaks under the "2/5" problem-transition badge —
  challenge is deferred ~2.3s on problem advance.
- Dropped the post-correct "There were N!" narration; trimmed the per-problem
  "Watch carefully!" so cadence doesn't feel naggy. Intro voice plays only
  on problem 0 of each level.

### Adventure-level voice cadence
- Voice playback rewritten to use the actual sound-completion callback (was
  a fixed-time gap that clipped longer clips). `playSequence(ids, gapAfterMs,
  onAllDone)` semantics — `gapAfterMs` is silence between clips, callback
  fires when the last one ends.
- Result narration order swapped to result-first, praise-second ("Five
  rockets!" → "Awesome!") — feels more like a teacher.
- Praise now fires ~40% of the time (Khan Academy Kids pattern); always
  speaks the result for educational reinforcement.
- Next problem doesn't advance until the praise voice finishes, then a
  1s breathing gap before the new instruction narrates.

### Problem transition between attempts
- ProblemTransition overlay: "Round N / M" card that pops in for ~2.2s
  between problems, with sparkle confetti falling from the top and a star
  sound effect at start.
- Ten-frame fades in (350ms) on each problem advance, active progress dot
  bounces in.
- Fixed: ten-frame no longer jumps vertically when feedback box renders
  (`submitArea` reserves minHeight).

### Addition / Subtraction narration
- Generated 300 new audio clips (`pre_have_<theme>_6..10` and
  `instr_add/sub_<theme>_5..9` across 10 themes) so the narrator always
  speaks the themed sentence ("You have 7 carrots! Add 5 more carrots!")
  instead of falling back to bare `num_N`.
- 5s inactivity → replay the current problem's instruction voice. Pairs
  with the existing 4s 👆 visual hint.

### Adventure navigation
- Exit-level now lands on the world's level-grid (one step back) instead
  of jumping all the way to the worlds grid. Reset is now parent-controlled
  via a `resetVersion` counter on AdventureMapScreen; bumped only on fresh
  entries (ModeChoice / sidebar Adventure button).
- 7th world card (Farm Share) needed bottom padding so it cleared the
  home-indicator safe area and was tappable.
- Voice cues for level transitions: `lvl_next` and `lvl_world_done`.

### Launch flow
- ModeChoice ("How do you want to play?") is now the first screen on a
  fresh install; PlayerSetup (theme/age/name) only fires after Free Play
  is picked. Adventure entry skips it since per-world themes apply.
- Welcome voice moved to ModeChoice (returning users still get it on
  app open).
- "Ten Frames" brand mark added to ModeChoice (mini ten-frame visual +
  wordmark).
- Space theme background image used for the ModeChoice screen with a soft
  dim gradient overlay.
- Boot splash (solid indigo) gates the main UI until persistence finishes
  loading, so free-play UI doesn't flash before ModeChoice / AdventureMap
  takes over. ModeChoice / PlayerSetup also render the backdrop while open.

### Workshop sandbox mode (earlier in the day)
- Creative free-play with a theme-emoji palette; tap to stamp/clear cells.
- Added to ModeSelector for both age groups; daily-limit-free.

### Voice timing fixes (cross-cutting)
- All `playSequence` callsites migrated to the new completion-callback
  semantics; gap parameter renamed `gapAfterMs`.
- `onRecordResult` is now driven by the voice's actual `onAllDone`, so the
  screen never advances mid-word.
- Inactivity replay shares the per-problem voice action via a ref
  (`lastInstructionVoiceRef`).
